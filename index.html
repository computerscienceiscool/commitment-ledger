commitment-ledger

commitment-ledger is a local-first PromiseGrid app prototype for tracking commitments against repository TODO work, preserving evidence, and recording assessments as signed CID-addressed artifacts.

Why teams use it

It separates available work from promises about that work, keeps the exact signed artifact bytes in local CAS, and gives operators one workflow for inspection, verification, exchange, and recovery.

One lifecycle, four layers

The tool is built around work observation, explicit promises, collected evidence, and separate human assessment.

Layer 1

scan

Observe branch-qualified TODO work from local repos without editing the source repos.

Layer 2

commit

Create commitments against discovered work targets and emit signed promise artifacts.

Layer 3

evidence

Record scan-derived or manual evidence while preserving commitment scope and artifact linkage.

Layer 4

assess

Turn evidence into explicit kept, broken, partial, delegated, or refused outcomes.

Operator surface

The repo is now more than a basic lifecycle CLI. It includes the operator views needed for local exchange, verification, and recovery.

Inspect

inspect + verify

Resolve artifact IDs, protocol docs, signer lineage, import provenance, and local trust status.

Exchange

export import send receive

Move bundles between repos, emit receive receipts, and preserve provenance for later audit.

Audit

status --exchange report --imports reconcile

Spot repeated imports, multi-source artifacts, receipt coverage, signer state, and trust results.

Recovery

doctor repair identity

Validate local state, classify repairable failures, and manage signer history, backup, restore, and rotation.

PromiseGrid shape

The implementation is local-first, but it follows the PromiseGrid discipline of frozen protocol docs, explicit conformance claims, and content-addressed artifacts.

Example commands

The CLI stays readable by default, but it also supports machine-readable JSON, local bundle exchange, and recovery-oriented workflows. Build the repo-local binary once, then these commands work as shown from the repo root.

examples 
make build
make demo-setup
./bin/commitment-ledger scan --config config/repos.demo.json
./bin/commitment-ledger commit --promiser Alice --repo alice-demo --branch main --target alice-demo/main/TODO-ravud/1 --due 2026-07-01 --promise "I promise to complete TODO-ravud subtask 1."
./bin/commitment-ledger inspect --json COMMITMENT-...
./bin/commitment-ledger verify --json ASSESSMENT-...
./bin/commitment-ledger status --exchange --json
./bin/commitment-ledger report --imports --json
./bin/commitment-ledger reconcile --commitment COMMITMENT-... --json
./bin/commitment-ledger doctor --repairable
./bin/commitment-ledger repair --json --import-artifacts --import-support --identity-lineage
./bin/commitment-ledger identity backup --include-imported-support --out /safe/path/identities.json
